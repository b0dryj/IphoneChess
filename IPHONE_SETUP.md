# Установка на iPhone без локального сервера

Теперь приложение работает как статический PWA: вся логика шахмат выполняется в браузере, а партия хранится в `localStorage` самого iPhone.

Это значит:

- Python-сервер для самой игры больше не нужен
- компьютер не должен быть включен во время игры
- после установки на iPhone приложение может открываться как обычное PWA

## Важное ограничение

PWA нельзя установить на iPhone просто из папки с файлами. Для первой установки сайт все равно должен быть открыт по `HTTPS`.

Практический вариант для этого проекта: опубликовать статическую версию через GitHub Pages.

## Что уже подготовлено в проекте

- вся игра перенесена в клиентский JavaScript
- состояние партии сохраняется прямо на устройстве
- готова папка `docs/` для публикации
- есть скрипт `scripts/sync_static_to_docs.ps1`, который копирует актуальную PWA-версию в `docs/`
- относительные пути настроены так, чтобы приложение работало не только в корне домена, но и в GitHub Pages

## Что делать

### 1. Подготовить статическую версию

В PowerShell из папки проекта:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\sync_static_to_docs.ps1
```

Это обновит папку `docs/`, из которой приложение можно публиковать как статический сайт.

### 2. Загрузить проект на GitHub

1. Создайте новый репозиторий на GitHub.
2. Загрузите в него проект.
3. Убедитесь, что папка `docs/` попала в репозиторий.

## 3. Включить GitHub Pages

1. Откройте репозиторий на GitHub.
2. Перейдите в `Settings`.
3. Откройте раздел `Pages`.
4. В разделе публикации выберите:

```text
Source: Deploy from a branch
Branch: main
Folder: /docs
```

5. Сохраните настройки.
6. Подождите публикацию сайта.

После этого GitHub выдаст HTTPS-адрес сайта вида:

```text
https://<username>.github.io/<repository>/
```

## 4. Открыть на iPhone

1. На iPhone откройте `Safari`.
2. Перейдите по опубликованному HTTPS-адресу GitHub Pages.
3. Убедитесь, что сайт открылся корректно.

## 5. Установить на экран домой

1. В Safari нажмите `Поделиться`.
2. Выберите `На экран "Домой"`.
3. Если iPhone покажет `Open as Web App`, включите этот режим.
4. Нажмите `Добавить`.

После этого приложение появится на домашнем экране iPhone.

## Как это будет работать дальше

- приложение будет запускаться прямо на iPhone
- партия будет храниться локально на этом устройстве
- компьютер для игры больше не нужен
- интернет нужен только для первой загрузки и обновления версии

## Уведомления

В приложении есть кнопка `Тестовое уведомление`.

Для проверки уведомлений на компьютере:

1. Запустите локальный сервер:

```powershell
python main.py
```

2. Откройте `http://127.0.0.1:8000`.
3. Нажмите `Тестовое уведомление`.
4. Разрешите уведомления в браузере.
5. Системное уведомление должно появиться сразу после нажатия.

Если уведомление не появляется, проверьте разрешения сайта в браузере, системные настройки уведомлений Windows/macOS и режим `Не беспокоить`/`Фокусирование`.

Если нужен настоящий push при закрытом приложении, используйте не GitHub Pages, а Python backend из этого репозитория.

Что нужно дополнительно:

1. Установить зависимости:

```powershell
pip install -r requirements.txt
```

2. Сгенерировать VAPID-ключи:

```powershell
python .\scripts\generate_vapid_keys.py
```

3. Указать контакт для VAPID:

```powershell
set VAPID_SUBJECT=mailto:your-email@example.com
```

4. Запустить приложение по HTTPS через `main.py`.

Как это работает:

1. Откройте установленное приложение на iPhone.
2. Нажмите `Тестовое уведомление`.
3. Если iPhone запросит разрешение на уведомления, нажмите `Разрешить`.
4. Браузер подпишет устройство на push.
5. Сервер отправит настоящий Web Push через 10 секунд.
6. Уведомление должно прийти, даже если вы закроете приложение.

Ограничения:

- на iPhone web push работает только для установленного приложения с экрана домой
- нужен `HTTPS`
- нужен работающий backend, который хранит подписку и отправляет push
- GitHub Pages для этого режима не подходит, потому что там нет серверной части

## Как обновлять приложение

Если вы меняете интерфейс или код:

1. Снова выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\sync_static_to_docs.ps1
```

2. Отправьте изменения в GitHub.
3. Подождите обновление GitHub Pages.
4. Откройте сайт на iPhone еще раз, чтобы service worker забрал новую версию.

## Если приложение не ставится

- открывайте сайт именно в `Safari`, а не во встроенном браузере другого приложения
- убедитесь, что сайт открыт по `HTTPS`
- если пункта `На экран "Домой"` не видно, в меню `Поделиться` прокрутите вниз и добавьте его через `Edit Actions`

## Если нужен самый короткий сценарий

1. Запустите `.\scripts\sync_static_to_docs.ps1`
2. Загрузите проект в GitHub
3. Включите GitHub Pages из папки `/docs`
4. Откройте выданный HTTPS-адрес в Safari на iPhone
5. Нажмите `Поделиться` -> `На экран "Домой"` -> `Добавить`
