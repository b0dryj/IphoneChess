# Local Chess PWA

Локальное PWA-приложение для игры в шахматы на одном устройстве. Основной рантайм теперь полностью клиентский: партия хранится в браузере, а для самой игры сервер не нужен.

## iPhone

Подробная установка без локального сервера описана в `IPHONE_SETUP.md`.

Для подготовки статической версии выполните:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\sync_static_to_docs.ps1
```

После этого папка `docs/` готова для GitHub Pages или другого статического HTTPS-хостинга.

## Локальное превью

```bash
python main.py
```

Откройте `http://127.0.0.1:8000`.

На локальном адресе кнопка `Тестовое уведомление` запрашивает разрешение браузера и сразу показывает системное уведомление Windows/macOS. Если уведомление не появляется, проверьте разрешения сайта в браузере и системные уведомления ОС.

`main.py` теперь нужен в основном для локального просмотра в браузере и опционального Web Push backend. Для iPhone без сервера используйте публикацию из `docs/`.

## Что умеет

- локальная партия на двоих на одном устройстве
- сохранение партии на устройстве
- шахматная валидация полностью в браузере
- рокировка, взятие на проходе, автоматическое превращение пешки в ферзя
- проверка шаха, мата и пата
- тестовое системное уведомление на компьютере сразу после нажатия кнопки
- системное уведомление на компьютере, когда один игрок ставит мат другому
- установка как PWA и офлайн-кэш статических ресурсов
- готовая статическая версия в `docs/`

## Закрытый push

Если нужен настоящий push при закрытом приложении, GitHub Pages недостаточно. Для этого нужен запущенный Python backend с Web Push.

То есть:

- `docs/` и GitHub Pages подходят для обычной статической PWA
- для push при закрытом приложении нужно развертывать `main.py` на публичном HTTPS-хостинге

Базовая настройка:

```powershell
pip install -r requirements.txt
python .\scripts\generate_vapid_keys.py
set VAPID_SUBJECT=mailto:your-email@example.com
python main.py
```

Для iPhone это приложение должно быть установлено на экран домой и открыто по HTTPS. Подробности описаны в `IPHONE_SETUP.md`.
